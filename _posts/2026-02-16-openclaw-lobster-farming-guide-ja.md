### はじめに：旧正月は変わらず、今年はロブスターが一匹増えました

旧正月はいつもと同じ旧正月です。

しかし今年は新しいおもちゃ、OpenClawが増えました。

あの感覚、ご存知でしょうか？お気に入りフォルダには「後で見る」ものが山のように溜まり、普段は忙しくて手をつける暇もないのですが、休暇に入ると堰を切ったように、すべてを消化してしまいたくなります。今年の旧正月、私が最初に着手した「堰を切る」プロジェクトは、少し突飛に聞こえるかもしれませんが、**私のLenovo Legion GoゲーミングハンドヘルドPCをAI Agentサーバーに変身させる**という計画でした。

「ロブスターを飼育する」とはどういう意味でしょうか？OpenClawは最近非常に話題になっているオープンソースのAI Agentプラットフォームで、そのロゴはロブスターのハサミです。そのため、界隈では自分のデバイスでOpenClawを稼働させることを「ロブスター飼育」と呼んでいます。私がやろうとしたのは、本来『バルダーズ・ゲート3』をプレイするために使っていたハンドヘルドPCを、Feishuに住むロブスターに変えることでした。

突飛に聞こえるかもしれませんが、実際にやってみると、本当に30分もかかりませんでした。

しかも、飼い始めてから2日も経たないうちに、OpenAIがOpenClawの創設者を「買収」しました。ちょうど自分で触っていたところだったので、ニュースを見た瞬間、「ああ、なるほど」と思いました。

この記事は、このロブスターの話から始まります。

### 第一幕：Windowsの「水が合わない」状況 — Mac miniが飛ぶように売れたのも納得です

![Windowsの混乱 vs Macの秩序](pic/2026-02-16-openclaw-lobster-farming-guide_02_ai.png)

まず、「事件現場」について説明します。

私はLenovo Legion Goを持っています。ご存じない方のために簡単に説明すると、これはLenovoがリリースしたWindowsゲーミングハンドヘルドPCで、8.8インチの大画面、AMDプロセッサー、16GBメモリを搭載しており、コントローラーは取り外し可能で、ディスプレイに接続すればmini PCとしても機能します。Nintendo Switchに似ていますが、完全なWindows 11が動作します。もちろん、SteamOS版もあります。（これは広告ではありません、笑）

正直なところ、このハンドヘルドPCを購入してから、ほとんどゲームはしていません。ずっとホコリをかぶっていました。それを見て、「キノコが生えるほど放置するくらいなら、仕事させよう」と思いました。つまり、AI Agentを動かして、私の24時間365日対応のプライベートアシスタントにしようと。

そして、Windowsの「水が合わない」という状態を痛感することになりました。

Windows上でNode.jsのバックグラウンドサービスを動かそうとすると、何に直面するでしょうか？

Windows Defenderがプロセスをウイルスとして停止させたり、PowerShellとcmdの2つのコマンドラインが互いに干渉し合ったり、深夜3時にシステムが自動更新で再起動したり、ファイルパスのバックスラッシュとスラッシュが常に衝突したりします。さらに、「このLinuxツールはWindowsではサポートされていません」といった様々な問題もあります。

**Mac mini M4が登場するやいなや品切れになるほど人気が出たのも納得です。**

Macがどれほど優れているかというよりは、Windowsで様々なコーディングCLIやOpenClawを動かすことが、本当にうんざりするような体験なのです。Macユーザーなら？ターミナルを開いて、brewをインストールすれば終わり。一言で言えば「楽」。Windowsは？一言で言えば「手間がかかる」。

幸いなことに、Windowsには救いの綱がありました。それが**WSL**です。

### 第二幕：Dockerの「理想」とWSLの「現実」

私の当初の計画はDockerを使うことでした。

何と言っても、Dockerは「標準的なやり方」ですからね。コンテナ化されたデプロイメント、環境隔離、ワンクリック起動など、

現実には、Legion GoでDocker Desktopを動かすのは、体重100kgの人が満員電車に乗り込むようなものでした。技術的には可能ですが、体験としては最悪です。Windows 11自体が4-5GBのメモリを消費し、Docker Desktopがさらに2-3GB消費すると、16GBのメモリの半分近くが消え、OpenClawに残されるメモリは施しのようなものでした。その上、途中で様々な小さな問題に遭遇し、各ステップで長い「待ち時間」が発生するのも言うまでもありません。

Dockerと約45分格闘した後、私は現実的な決断を下しました。

**もういいや。**

結局のところ、これは自分用のものですし、顧客への納品ではありません。わざわざ自分を追い込む必要はないでしょう？

最終的に私が選んだのは、最もシンプルな方法でした。WSL Ubuntu上でOpenClawを直接稼働させる、というものです。入れ子構造は避け、Linux一層のみという、単純かつ手荒いやり方です。

結果として、これが正解でした。ゼロから始めてFeishuでロブスターからの最初の返信を受け取るまで、**合計30分もかかりませんでした**。

（具体的なチュートリアルは記事末尾の参考部分に記載していますので、ご自身で試したい方は直接最後までスクロールしてご覧ください。）

Feishuでロブスターが最初のメッセージに返信したのを見たとき、正直なところ、かなり達成感がありました。ホコリをかぶっていたゲーミングハンドヘルドPC+オープンソースプロジェクト+チャットソフトウェア、これらを組み合わせてプライベートAIアシスタントを作り出したのです。この感覚、何と表現すればいいでしょうか。レゴで走る車を組み立てたような気分です。

### 第三幕：ロブスターに安価な「脳」を見つける

OpenClawのインストールも終わり、Feishuとの連携もできました。次に最も重要な問題です。**どのLLMをロブスターの「脳」として使うか？**

OpenClawの公式推奨はClaude Opus 4.5です。その効果は確かに申し分ありません。しかし、私には非常に現実的な問題がありました。

**私は中国国内にいて、VPN（科学上网）のリソースが限られているのです。**

具体的に言うと、VPNは同時に1つのPC端末でしか使えません。日中の仕事用のMacはVPNに接続しておく必要があり、「遊びで飼育する」ハンドヘルドPCにVPNを譲るわけにはいきません。そしてLegion GoでOpenClawを稼働させるのは24時間365日の話です。

たかがロブスターのために、仕事用のマシンをネットから切断するわけにはいかないでしょう？

これが現実的な選択を迫りました。国内の大規模言語モデルを使うことです。

幸いなことに、この2年間で国内の大規模言語モデルのコストパフォーマンスは確かに向上しました。私が最終的に選んだのは**Kimi K2.5**で、月額199元のKimi for Codingプランを購入しました。

なぜそれを選んだのか？理由はシンプルです。

第一に、国内から直接接続でき、VPNが不要で遅延が少ないこと。第二に、月額199元は、「遊びで飼育する」ロブスターには十分すぎるほどです。第三に、以前AIコーディングAgentの比較レビューでKimiをテストした際、明確な要件の下では、このモデルが非常に手際が良いことを確認していました。

もちろん、KimiとClaudeにはまだ差があります。以前書いたAIプログラミングアシスタントの比較レビューでは、Kimiは「会議で最初に手を挙げて発言するが、誰かが『それは違うと思う』と言うとすぐに意見を変える」タイプでした。しかし、日常的なアシスタントの脳としてなら十分です。ロブスターが首席アーキテクトになる必要はなく、資料検索や情報整理、簡単な質問に答えてくれるだけで十分なのです。

**月額199元の「脳」で24時間365日オンラインのAIアシスタントを稼働させる。コストパフォーマンスは完璧です。**

![LLMの脳の選択：ロブスターに安価な脳を見つける](pic/2026-02-16-openclaw-lobster-farming-guide_06_html.png)

ここまで来て、やはり感嘆せざるを得ません。VPNの制限は確かに生産性にとって障害です。同じことでも、海外のユーザーならClaudeのAPIキーを設定すれば済むところを、国内では大きく迂回しなければなりません。しかし、別の視点から見ると、この制限があったからこそ「国産大規模言語モデルを強制的に体験する」機会が得られたとも言えます。禍を転じて福と為す、といったところでしょうか？

### 第四幕：「裸の機械」から「使えるもの」へ — インストールすれば万能だと思っていましたか？

ロブスターを飼育し、Feishuとの連携もでき、脳も接続しました。

それで？

その結果、私は気づいたのです。**これは「裸の機械」に過ぎない、と。**

このギャップを味わってみてください。ネット上のデモンストレーションビデオでは、OpenClawは航空券の予約、コードの記述、スケジュール管理、データ分析など、何でもこなしているように見え、まるでアイアンマンのジャービスのようでした。期待に胸を膨らませてインストールを終え、意気揚々とFeishuで「明日の東京の天気調べて」と尋ねると、味気ないテキストが返ってきて、APIを直接呼び出すのと大差ありません。

**ジャービス？むしろSiriのようでした。**

これはまるで、ハイスペックなPCを組み立てたものの、起動してみるとデスクトップにはごみ箱とメモ帳しかなく、思い描いていた「生産性モンスター」とは程遠い、という感覚に似ています。

何が足りないのか？それは二つの要素です。**「行動様式」と「スキル」。**

![OpenClawの2つの状態：裸の機械 vs フル装備](pic/2026-02-16-openclaw-lobster-farming-guide_03_html.png)

#### ロブスターに「憲法」を制定する

OpenClawには非常に洗練された設計があります。その個性、ルール、行動の境界はすべてMarkdownファイルで定義されます。

そうです、.mdファイルです。コードを書く必要も、パラメーターを調整する必要もなく、自然言語で「どのように仕事をするべきか」を伝えるだけで良いのです。

主要なファイルは以下の通りです。

- `AGENTS.md`——ロブスターの「憲法」に相当し、その能力の範囲と行動の制約を定義します。
- `SOUL.md`——その人格を定義し、話し方がクールなのか、おしゃべりなのかなどを設定します。
- `MEMORY.md`——ロブスターの「ノート」で、物事を記憶するために自分自身で書き込むことができます。

私はAGENTS.mdにいくつかの重要なルールを記述しました。返信は中国語で行うこと、指示されていないことを自発的に行わないこと、金銭、メッセージ送信、ファイル削除に関わる操作は必ず先に私に確認すること、情報整理は長い文章ではなく表を用いること、といった内容です。

非常にシンプルなルールに見えますが、これらのルールがロブスターとの「信頼関係」を決定します。定義が明確であればあるほど、その行動は予測可能になり、より安心して仕事を任せることができます。**人との協業と同じで、期待が曖昧であればあるほど、相手からの成果はランダムになります。**

#### ロブスターに「App Store」を導入する

憲法だけでは不十分です。何のスキルもインストールされていないOpenClawは、アプリが入っていないiPhoneのようなもので、電話はできますが、それしかできません。

OpenClawのスキルエコシステムは、2026年2月時点で、**5700以上のコミュニティによって構築されたスキル**があり、27の主要カテゴリをカバーしています。Google Workspace管理、ブラウザ自動化、GitHub統合からマーケティングツール、CRM管理、ウェブスクレイピングまで、あらゆるものがあります。インストールはコマンド1行で済みます。

ただし、ここで非常に重要なセキュリティ問題について触れておく必要があります。

2026年2月、セキュリティ研究機関はClawHub上でマルウェアやキーロガーを配布する**341の悪意あるスキル**を発見しました。別のセキュリティ企業は、約7%のスキルにAPIキーの漏洩問題があることを確認しています。

考えてみてください、これは初期のAndroidアプリストアと全く同じではないでしょうか？エコシステムが繁栄する裏側には、玉石混淆の状況があります。そのため、サードパーティ製のスキルをインストールする前に、必ずソースコードを確認し、セキュリティスキャン結果を調べ、公式の名前空間のスキルを優先的に利用するようにしてください。インストールするすべてのスキルは、Agentの完全な実行権限を取得する可能性があるからです。

（おすすめスキルのリストとインストール方法も、記事末尾の参考部分に記載しています。）

### 第五幕：ロブスターを飼い始めた途端、OpenAIが「人材確保」に乗り出しました

ロブスターを飼い始めて2日も経たないうちに、私は大きなニュースを目にしました。

**OpenClawの創設者であるPeter Steinberger氏がOpenAIへの参加を発表したのです。**

正確に言えば、これは伝統的な意味での「買収」ではありません。OpenClawがOpenAIのプライベート製品になるわけではなく、独立したオープンソース財団に移管され、MITライセンスの下でオープンソースとして継続されます。しかし、Steinberger氏自身はOpenAIに入社し、「次世代のパーソナルAgentの推進」を担当します。Sam Altman氏自身が投稿で確認し、彼を「天才」と呼び、この仕事が「我々の製品の核として急速に位置づけられるだろう」と述べました。

36Krの報道によると、決定を下す前に、**MetaとOpenAIが同時に彼を争奪していた**とのことです。Steinberger氏が最終的にOpenAIを選んだ理由は、「OpenAIと協力することが、Agentをすべての人に届ける最速の道である」というものでした。

すごいですね。

GitHubで14.5万スター。リリースから巨大企業に争奪されるまで、わずか4ヶ月足らずです。

このニュースを見たとき、私はちょうど旧正月を丸々使ってOpenClawをいじくり回し、自分でインストールし、設定し、飼育し、問題に遭遇していたため、この取引のロジックを瞬時に理解しました。

**OpenAIが注目したのはOpenClawという製品ではなく、OpenClawが検証した「アーキテクチャ」なのです。**

### 分解して見る：ロブスターの構造とは？

いじくり回した後で振り返ってみると、OpenClawのアーキテクチャは実は4つの要素に分解できることがわかります。

**第一に、脳（LLM）。** どのような大規模言語モデルでも接続可能です。Claude、GPT、Gemini、Kimi、さらにはローカルで実行されるオープンソースモデルも含まれます。脳は交換可能です。今日はKimiを使い、明日モデルが更新されればClaudeに切り替えればよいのです。ロブスターの「知能」はいつでもアップグレードできます。

**第二に、永続的なサンドボックス。** これがOpenClawとこれまでのすべてのAI Agentソリューションとの最大の違いです。

以前のAgent、AutoGPTであれ、様々な「AIアシスタント」であれ、どれも「一時的な作業員」モードでした。タスクを与えると、一時的に環境を立ち上げて実行し、終われば消滅します。次回使うときは？すべて最初からやり直しです。

OpenClawは違います。その環境は**常に存在しています**。Agentは独自のファイルシステム、記憶データベース、会話履歴、インストール済みのツールを持っています。「使い捨て」の臨時作業員ではなく、「そこに常にいる」常駐従業員なのです。

この違いをよく考えてみてください。

「常にそこにいる」Agentは、記憶を蓄積し、使えば使うほどあなたのことを理解します。バックグラウンドで定時タスクを実行でき、毎回手動でトリガーする必要はありません。自身の作業状態を維持し、中断しても続きから作業できます。

これが、「ロブスター飼育」の体験が、以前のどのAIツールとも違うと私が言った理由です。**それは賢くなったのではなく、より「存在している」のです。**

**第三に、行動様式（.md設定ファイル）。** 自然言語でMarkdownファイルをいくつか書くだけで、Agentの人格、能力の境界、行動規範を定義できます。コードを書く必要はありません。この設計の素晴らしい点は、AIの「設定」を「説明書を書く」のと同じくらい簡単にしたことです。文字が書ければ、自分自身のAgentをカスタマイズできるのです。

**第四に、オープンなスキルエコシステム。** モジュール化された能力パックで、コミュニティが共同で構築し、必要に応じてインストールできます。スマートフォンのApp Storeのようなものです。

キーワードは**「オープン」**です。以前のAgentソリューションのほとんどはクローズドでした。機能はプラットフォームに組み込まれており、ユーザーはプラットフォームが提供する機能しか利用できませんでした。OpenClawはスキルをコミュニティに公開しました。その結果は？半年足らずで、5700以上のスキルが生まれました。

エコシステムが与えられれば、進化は速い。これはインターネットが20年間で繰り返し検証してきた真理です。

**「脳 + 永続的なサンドボックス + 行動様式設定 + オープンなスキルエコシステム」——この4点セットの組み合わせは、将来的に個人AIアシスタント、さらには企業AIアシスタントの標準形態となる可能性が非常に高いでしょう。**

![AI Agent「4点セット」標準形態](pic/2026-02-16-openclaw-lobster-farming-guide_04_html.png)

これまでのソリューションと比較すると、主な違いは2点です。
1. サンドボックス環境が常に存在し、タスクに応じて一時的に生成されるものではない
2. スキルがエコシステムに開放されており、プラットフォームが独自に開発したものではない

OpenAIが見ていたのは、この点です。彼らが買収したのは製品ではなく、**14.5万個のスターによって検証されたアーキテクチャの思想、そしてそのアーキテクチャを創造した人物なのです。**

### では、私たちは何をすべきでしょうか？

もう少し現実的な話をしましょう。

もしあなたがAIの方向性を考えているなら、この状況は非常に明確です。**汎用型AIアシスタントや日常の雑事を解決する中小規模のプロジェクトには、大きなチャンスはもうないかもしれません。**

OpenAIがOpenClawのアーキテクチャを自社の製品ライン、おそらくChatGPTの次期メジャーバージョンに統合すれば、そのモデル能力、ユーザー基盤、流通チャネルを考慮すると、小規模チームに残される余地は非常に限られます。半年間苦労して開発した「AIがフードデリバリーを注文+スケジュール管理+メール作成」といった機能は、一夜にしてGPTのネイティブ機能に置き換えられてしまうかもしれません。

しかし、チャンスがないわけではありません。別の方向にあるのです。

**この「4点セット」アーキテクチャを中心に、toB（企業向け）やより垂直な分野のシナリオを展開することです。**

考えてみてください。企業顧客が求めているのは「何でもできる」ことではなく、「私のビジネスシナリオにおいて、安全かつコンプライアンスに準拠しながらこの一つの仕事をきちんとこなす」ことです。権限管理、データセキュリティ、コンプライアンス監査といったものは、OpenAIやいかなる汎用プラットフォームでも対応しきれるものではなく、深い業界理解とカスタマイズ能力が必要です。

そして、OpenClawが検証したこのアーキテクチャは、まさに優れた「基盤」となります。ゼロから車輪を再発明する必要はなく、この基盤の上で垂直なカスタマイズを行えば良いのです。

簡単に言えば、**基礎はすでに誰かが作ってくれたので、あなたは自分の分野の建物をその上に建てるだけでいい、ということです。**

![垂直化の機会：オープンな基盤の上に自分たちの建物を建てる](pic/2026-02-16-openclaw-lobster-farming-guide_07_html.png)

### 結び：ロブスター飼育から得られた二つの考察

旧正月を丸々費やして、ロブスターを一匹飼育し、ついでに二つのことを悟りました。

**一つ目：今後、誰もが自分専用の「ロブスター」を持つようになるかもしれません。**

OpenClawの「一人+一匹のロブスター」モデルは、本質的に一人の人間を「小規模チーム」に変えるものです。一人+カスタマイズされたAgent+インストール済みのスキル = 以前なら数人が必要だったかもしれないことをカバーできる、ということです。

さらに大きく考えてみましょう。もし会社の一人ひとりが自分専用のロブスターを持っていたらどうでしょうか？それは汎用的なChatGPTの入り口ではなく、OpenClawのように永続的な環境を持ち、カスタマイズされた設定を持ち、具体的なビジネスに関連するスキルを備えた専用のAgentです。

**一人 + 自分専用のAgent = 「スーパー個人」。**

これはSFではありません。数日間ロブスターを飼育した後、実際に自分が処理できる情報量が増えたと感じました。もしチームの全員がそれぞれ一匹ずつ持っていたら、できることはかなり増えるでしょう。

**二つ目：今後、企業は「壊しても大丈夫」なビジネスの遊び場を作る必要があるかもしれません。**

OpenClawのサンドボックス設計は私に大きなインスピレーションを与えました。Agentが「安全でありながらも実際の操作を実行できる」環境で作業できるようにするものです。壊してしまっても？大丈夫、サンドボックスが対応してくれます。

企業もまた、このような「サンドボックス」を必要としています。Agentがナレッジベース、CRM、プロジェクト管理システムにアクセスできるものの、明確な権限境界があり、監査ログやロールバックメカニズムがあること。もし失敗しても元に戻せることです。

今後、企業のAIレベルを区別するのは、「AIを使っています」という点ではありません。ChatGPTは誰でも使えますから。そうではなく、「私たちはAIのために安全なビジネス環境を構築し、AIが質問に答えるだけでなく、実際にビジネスプロセスに参加できるようにしている」という点でしょう。

**これこそが、真のAIネイティブ組織のあるべき姿なのかもしれません。**

---

私のロブスターの話に戻ります。

私のロブスターは今、Legion Go上で静かに稼働しており、Feishuに接続されています。時々何かを調べたり、情報を整理したりするのを手伝ってくれます。それほど賢いわけではありませんが、やはり「脳」は199元のKimiですから。しかし、**常にそこにいる**という点が勝っています。

時には深夜にふと何かを思いつき、携帯電話でFeishuにメッセージを送ると、すぐに返信が来ます。PCを開く必要も、ウェブサイトにログインする必要も、これまでの経緯を最初から説明する必要もありません。なぜなら、それが記憶しているからです。

この体験は、「ブラウザを開く→chatgpt.comと入力→ログイン→新規会話開始→最初から背景を説明」とは全く異なります。

**まるで眠らないパートナーのようです。最も賢いわけではないかもしれませんが、常にオンラインなのです。**

私はますます、これこそがAI Agentの真の方向性ではないかと感じています。賢くなることではなく、**より「その場に存在すること」**です。

旧正月は終わりましたが、ロブスターはまだ稼働しています。

## 参考

### 付録一：WSL Ubuntu + OpenClaw + Feishu 完全デプロイチュートリアル（以下の内容はAIに再検索・整理させたものです。当時、私自身も操作の詳細を少し忘れてしまっていたためです）

**システム要件**：Windows 10/11（64ビット）、管理者権限、安定したネットワーク

#### ステップ1：WSL2とUbuntuのインストール

PowerShell（管理者モード）を開きます。

```powershell
wsl --install -d Ubuntu-24.04
```

インストール完了後、Windowsを再起動します。スタートメニューからUbuntuを開き、Linuxのユーザー名とパスワードを設定します。

#### ステップ2：systemdの有効化

OpenClawのデーモンプロセスにはsystemdが必要です。Ubuntuターミナルで実行します。

```bash
sudo tee /etc/wsl.conf >/dev/null <<'EOF'
[boot]
systemd=true
EOF
```

PowerShellに戻り、WSLをシャットダウンしてから再度開きます。

```powershell
wsl --shutdown
```

#### ステップ3：Node.js 22のインストール

方法1（推奨）——nvmを使用：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 22
nvm use 22
node -v    # v22.x.x と表示されるはずです
```

方法2——NodeSourceから直接インストール：

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

#### ステップ4：OpenClawのインストール

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

セットアップウィザードがGateway設定、AIモデルのAPIキー、およびsystemdデーモンプロセスを構成します。

#### ステップ5：Feishuへの接続

Feishuプラグインをインストールします。

```bash
openclaw plugins install @openclaw/feishu
```

Feishuオープンプラットフォーム（`open.feishu.cn/app`）で企業自社アプリケーションを作成します。

1.  アプリケーションを作成し、**App ID** と **App Secret** をコピーします。
2.  権限を設定します: `im:message`、`im:message:send_as_bot`、`im:chat.members:bot_access`、`contact:user.employee_id:readonly`
3.  ボット機能を有効にします。
4.  イベントサブスクリプションで「ロングコネクションイベント受信（WebSocket）」を選択し、`im.message.receive_v1`を追加します。
5.  バージョンを作成 → 承認公開します。

OpenClawの接続を設定します。

```bash
openclaw channels add
# Feishuを選択し、App IDとApp Secretを貼り付けます
```

または、`~/.openclaw/openclaw.json`を手動で編集します。

```json5
{
  channels: {
    feishu: {
      enabled: true,
      dmPolicy: "pairing",
      accounts: {
        main: {
          appId: "cli_あなたのAppID",
          appSecret: "あなたのAppSecret",
          botName: "私のAIロブスター",
        },
      },
    },
  },
}
```

起動してテストします。

```bash
openclaw gateway

# 別のターミナルでペアリングを承認
openclaw pairing list feishu
openclaw pairing approve feishu <ペアリングコード>
```

#### 重要な落とし穴

**ファイルパス**：`~/.openclaw/`は必ずWSLのLinuxファイルシステム内（`/home/あなたのユーザー名/`）に配置してください。`/mnt/c/`の下に置くと、IO性能が著しく低下し、ロブスターの動作が非常に遅くなります。

### 付録三：OpenClaw主要.md設定ファイルの説明

| ファイル | 役割 | 推奨事項 |
|------|------|------|
| `AGENTS.md` | コアな実行指示、能力の境界と行動制約を定義 | 真剣に記述すること、これはロブスターの「憲法」です |
| `SOUL.md` | 人格とインタラクションスタイル | 自分の好みに合わせて、ただしあまりにも「中二病」にならないように |
| `TOOLS.md` | ユーザー定義ツール仕様とメモ | 必要に応じて追加 |
| `MEMORY.md` | 構造化された記憶、Agentが自律的に読み書き可能 | ロブスター自身にメンテナンスさせる |

設定ファイルは`~/.openclaw/openclaw.json`（JSON5形式）にあり、OpenAI API形式と互換性のあるあらゆるモデルプロバイダーをサポートしています。
